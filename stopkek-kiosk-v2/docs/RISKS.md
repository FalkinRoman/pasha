# Подложка StopKEK — риски и как они закрыты

Каждый риск → конкретное место в коде/настройке, которое его закрывает. Это чеклист
приёмки Фазы 6 (стресс-тест взлома).

## Обход блокировки / «бесплатная игра»

| Атака игрока | Закрыто чем |
|---|---|
| Снять оверлей через Диспетчер задач | `DisableTaskMgr` (`02-apply-policies.ps1`) + стандартный аккаунт не может убить службу SYSTEM |
| Убить процесс shell | `ShellWatchdog` перезапускает + лочит ОС на время отсутствия (Фаза 2) |
| Остановить/удалить службу | Стандартный аккаунт без прав на SCM; служба `obj= LocalSystem` |
| Служба упала сама | `sc failure … restart/5000` ×3 (`03-install-service.ps1`) |
| `Win+L` / уйти на экран Windows | `DisableLockWorkstation=1` |
| Сменить пользователя / выйти | `HideFastUserSwitching=1`, `NoLogoff=1` |
| `regedit` снять политики | `DisableRegistryTools=1` |
| `Run`/`cmd` запустить своё | `NoRun=1` + AppLocker whitelist (Фаза 0b) |
| Перевести системные часы → больше времени | Стандартный аккаунт без права смены времени; **монотонный таймер** в `SessionStateMachine.ComputeRemaining` не зависит от wall-clock |
| Alt+F4 по замку | Хук клавиатуры в LOCKED (Фаза 2) + WM_CLOSE игнор |
| Принести свой `.exe`/игру | AppLocker/SRP whitelist (Фаза 0b) |

## Сетевые / временные сбои

| Ситуация | Поведение | Где |
|---|---|---|
| Сеть упала во время игры | доигрывает по локальному `endAt`, не лочим | `ApplyOffline` → Active keep-alive |
| Сеть упала в LOCKED | остаёмся LOCKED (fail-secure) | `ApplyOffline` → Locked |
| Сервер вернул один глюк `locked` в активной сессии | держим сессию (гистерезис ×2) | `EnterLockedWithHysteresis` |
| Таймаут/битый JSON | не падаем, трактуем как offline/online-ошибку | `KioskApiClient.GetStateAsync` → `PollResult` |
| Нет config.json / пустой kioskKey | служба не стартует «вхолостую разблокированной», падает громко | `Program.cs` + `KioskConfig.Validate` |
| Перезагрузка/свет | автологин + автостарт службы → состояние восстановлено с сервера (`pcUnlockedAt`/`endAt`) | Фаза 0 + опрос |

## UX «не мешать игре»

| Требование | Решение | Где |
|---|---|---|
| В игре оверлея нет | Mode=Active → shell прячет всё, кроме виджета | `KioskView.Mode` (Фаза 2) |
| Предупреждения не вырывают из катки | серверный `notice` 15/5/1 → тост, не модал | `KioskView.Notice` |
| Время вышло — дать доиграть | GRACE на `graceSeconds`, потом мягкий relock | `EnterGrace` / `ApplyOfflineGrace` |
| Игрок сам закрывает сессию | команда `end-session` из виджета → `POST /kiosk/end-session` | `Worker.HandleCommandAsync` |
| Мультимонитор без щелей | окно-замок на каждый дисплей | Фаза 2 |
| Exclusive-fullscreen игра поверх замка | свернуть foreground перед relock | Фаза 2 |

## Чеклист приёмки (Фаза 6)

- [ ] Диспетчер задач недоступен под игроком
- [ ] kill shell → ОС лочится и shell поднимается < 2с
- [ ] попытка остановить службу под игроком → отказ
- [ ] обрыв сети в игре → доигрывает до endAt, затем грейс → замок
- [ ] обрыв сети в LOCKED → остаётся замок
- [ ] перевод часов вперёд → время не прибавилось
- [ ] reboot во время сессии → состояние корректно восстановлено
- [ ] мультимонитор → замок на всех экранах
- [ ] Alt+F4 / Win / Alt+Tab в LOCKED → подавлены
- [ ] юнит-тесты `dotnet test` зелёные
