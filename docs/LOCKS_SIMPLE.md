# Замки stopkek — простое решение

## Задача

Клиент в приложении нажимает **«Открыть главную дверь»** → дверь открывается на несколько секунд. Команда идёт **с сервера stopkek**, только в окне оплаченной брони. Замков на боксы/капсулы нет.

---

## Что покупаем (5 дверей)

| № | Что | Кол-во | Зачем |
|---|-----|--------|--------|
| 1 | Электрозамок 12V, 350 кг (например R-Lock RL-EM350-LSS) | 1 | Главный вход |
| 2 | Блок питания 12V 3A | 1 | Питание замка |
| 3 | **Shelly Plus 1** (Wi‑Fi реле) | 1 | «Кнопка открыть» по команде из сети |
| 4 | Raspberry Pi 4 (или старый мини‑ПК) | 1 | Мост в клубе |
| 5 | Роутер Wi‑Fi 2.4 GHz | 1 | Если в клубе ещё нет |

**Не покупаем:** терминалы с лицом (F-KD iFlow) и Tuya-панели (CF3) — с нашим приложением не стыкуются без лишней возни.

---

## Как это работает

```
Телефон → stopkek API (VPS) → мост в клубе (Pi) → Shelly → замок
```

1. Клиент жмёт кнопку в мобилке.
2. Сервер проверяет бронь и шлёт команду на **мост**.
3. Мост по Wi‑Fi включает нужное **реле Shelly** на 3–5 секунд.
4. Замок отпускает дверь.

**С ноутбука разработчика замки не открываются** — только через сервер и приложение.

---

## Что монтирует электрик

- Замок на дверь + якорь.
- БП 12V → замок.
- Shelly в разрыв цепи 12V (или через сухой контакт — по схеме монтажника).
- Всё в одной Wi‑Fi сети клуба.

Имена в системе (как в админке):

| lockId | Дверь |
|--------|--------|
| `main-door` | Главный вход |

---

## Что пишем мы (один раз)

**Мост** — маленькая программа на Pi в клубе (~100 строк):

- Принимает `POST /open` с `{ "lockId": "cell-2" }` и токеном (как уже ждёт API в `LocksService`).
- По таблице `lockId → IP Shelly` дергает локальный HTTP Shelly.
- Реле включается на 3–5 сек и выключается.

**В stopkek-api** менять почти ничего не нужно — в админке: **Замок**:

- режим **Боевой**
- `lockHttpBaseUrl`: адрес моста (через VPN, например Tailscale)
- токен моста
- ID: `main-door`

**Сеть:** Pi и VPS в одной VPN (Tailscale) — проще всего, без белого IP в клубе.

---

## Порядок запуска

1. Купить железо по списку выше.
2. Смонтировать 1 дверь → проверить руками из приложения Shelly.
3. Поставить Pi, написать/задеплоить мост, прописать IP Shelly в конфиге.
4. В админке stopkek — URL моста и `main-door`.
5. **Тест без брони:** Замок → «Тест: открыть дверь» (журнал ниже с пагинацией).
6. Тест из приложения: оплата → «Открыть главную дверь».

Пока железа нет — режим **Симуляция**. Мобилка уже ходит в тот же API; при смене на **Боевой** ребилд аппа не нужен.

---

## Тест без замков (сейчас)

В админке `lockProvider: mock` — в журнале пишется «открыто», замок не щёлкает. Удобно для разработки мобилки и API.
