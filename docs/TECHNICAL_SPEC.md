## 1. Введение

### 1.1 Название и суть

**stopkek** — экосистема для компьютерного клуба:

| Продукт | Назначение |
|---------|------------|
| **Mobile App** (iOS 14+, Android 8.0+) | Клиент: бронь, баланс, двери, таймер, push |
| **Backend API** | REST + WebSocket, бизнес-логика, интеграции |
| **PC Launcher** (Electron kiosk) | Заглушка на ПК клуба: вход, игры, лаунчеры, тех-чат |
| **Admin Panel** (Web) | Управление ПК, статусы, аналитика, удалённый доступ |
| **Telegram Bot** | Алерты админам, поддержка пользователей |

### 1.2 Цели

- Клиенты бронируют места, пополняют баланс, открывают двери/ячейки, продлевают сеанс.
- Администраторы управляют статусами ПК, смотрят аналитику, помогают удалённо.
- На ПК клуба — минимальный kiosk: авторизация, игры/платформы, Discord, тех-чат.

## 3. User Stories

### Клиент

1. Регистрация по телефону (SMS), опционально верификация документа.
2. Карта зала в реальном времени: свободен / занят / ремонт.
3. Бронь: место + время начала + длительность → оплата.
4. Пополнение баланса (карта, СБП).
5. Таймер сеанса + продление из приложения.
6. Push: начало/конец сеанса, низкий баланс.
7. Открытие главной двери и ячейки своего ПК на время активной брони.
8. Приёмка: чеклист периферии после прихода в ячейку.
9. Профиль: данные, баланс, история посещений и транзакций.
10. Обратная связь: отзывы и предложения.

### Администратор

1. Статусы ПК: свободен / занят / ремонт.
2. Статистика: посещаемость, доходы, воронка бронирования.
3. Удалённый доступ к ПК: перезагрузка, базовое управление (VNC/RDP — по возможности инфраструктуры).
4. Telegram-алерты: критические события, обращения в поддержку.

### ПК клуба (kiosk)

1. Вход в личный аккаунт (QR / PIN с мобилки).
2. Раздел «Игры» и «Игровые платформы» (Steam, Epic, Discord, тех-чат).
3. Автоблокировка при окончании сеанса.

## 4. Функциональные требования (MVP)

### 4.1 Регистрация и авторизация

- Вход по номеру телефона + SMS OTP (шлюз: SMS Aero или аналог).
- JWT access + refresh; сессии в Redis.
- Опционально (фаза 2 ТЗ): загрузка/проверка паспорта или иного документа — статус `verified` / `pending` / `rejected`.
- Профиль: имя, телефон, email (опц.), аватар.

### 4.2 Карта зала

- Интерактивная SVG/Canvas-схема (референс: floor-plan-reference).
- Зоны с метаданными: название, GPU, монитор, Hz, цена/час.
- Статусы мест:
  - 🟢 свободен
  - 🔴 занят
  - 🟡 забронирован (ещё не начался сеанс)
  - ⚫ ремонт / недоступен
- Real-time: WebSocket (Socket.io) или Firebase RTDB — **в смете: WebSocket на бэке**.
- Zoom, pan, центрирование; тап по месту → детали + «Забронировать».

**Модель данных (черновик):**

```
Club → Zone → Seat (computer)
Seat: { id, number, zoneId, status, hardwareSpec, lockId?, cellLockId? }
Zone: { id, name, pricePerHour, specs: { gpu, monitor, refreshRate } }
```

### 4.3 Бронирование

- Выбор места(мест), интервал `[start, end]`, расчёт тарифа по зоне и длительности.
- Статусы брони: `draft` → `pending_payment` → `paid` → `active` → `completed` | `cancelled` | `no_show`.
- Конфликты: двойная бронь на одно место запрещена (транзакция БД).
- История бронирований в профиле.

### 4.4 Баланс и оплата

- Внутренний кошелёк пользователя.
- Пополнение: YooKassa (+ СБП через YooKassa).
- Списание при оплате брони / продлении.
- История транзакций: тип, сумма, статус, внешний id платежа.
- Возврат неиспользованного времени при досрочном завершении — **нет**; продление в приложении. Ручные корректировки — через админку.

### 4.5 Таймер сеанса и продление

- При `active` брони — обратный отсчёт в приложении (синхронизация с сервером).
- Продление: выбор доп. времени → расчёт → оплата с баланса / картой.
- За N минут до конца — push «скоро конец сеанса».

### 4.6 Умные двери и ячейки

- На время оплаченной брони:
  - кнопка **«Открыть главную дверь»** (всегда доступна);
  - кнопка **«Открыть ячейку #N»** только для забронированного места N.
- API замков: MQTT и/или HTTP (конкретный вендор — уточняется на S-04).
- Логирование каждого открытия: userId, lockId, timestamp, result.
- Rate limit на открытие (защита от спама).

### 4.7 Приёмка клуба (чеклист)

- Триггер: пользователь отметил «Пришёл» / открыл ячейку / начался сеанс.
- Модалка/экран: галочки — монитор, мышь, наушники, клавиатура; комментарий; фото (опц., фаза 2).
- Сохранение в `acceptance_reports`; при расхождении — уведомление админу (TG).

### 4.8 Уведомления

- Push: FCM (Android) + APNs (iOS).
- События: начало сеанса, за 15/5 мин до конца, конец, успешная оплата, отклонение документа.

### 4.9 Обратная связь

- Форма: тема, текст, рейтинг (1–5), опц. скрин.
- Список обращений в админке; ответ через TG/email (MVP — просмотр в админке).

### 4.10 PC Launcher (заглушка → MVP)

- Electron, fullscreen kiosk, автозапуск Windows.
- Авторизация: QR (сессия с мобилки) или PIN.
- Экраны: главная, «Игры», «Платформы» (иконки Steam / Epic / Discord).
- Тех-чат: простой чат с админом (WebSocket → админка / TG).
- Блокировка UI при `session.ended`.
- Запуск лаунчеров — shell/deeplink где возможно; полная интеграция Steam API — не в MVP.

### 4.11 Admin Panel

- Дашборд: DAU, брони сегодня, выручка, занятость залов.
- CRUD мест и зон; смена статуса ПК вручную.
- Список броней, пользователей, транзакций.
- Просмотр acceptance reports.
- Удалённый доступ: reboot PC, базовые команды (зависит от агента на ПК).
- WebSocket live-статусы зала.

### 4.12 Telegram Bot

- Подписка админов на алерты.
- Команды: `/status`, `/bookings today`, критические ошибки замков.
- Пользовательская поддержка: пересылка сообщения в админ-чат.

## 6. Архитектура

```
┌─────────────┐     ┌─────────────┐     ┌──────────────┐
│ Mobile App  │────▶│  Backend    │◀────│ Admin Panel  │
│ RN + Expo   │ WS  │  NestJS     │ WS  │ React + Vite │
└─────────────┘     │  PostgreSQL │     └──────────────┘
       │            │  Redis      │            │
       │            └──────┬──────┘            │
       │                   │                   │
┌──────▼──────┐     ┌──────▼──────┐     ┌──────▼──────┐
│ PC Launcher │     │ Locks MQTT  │     │ Telegram    │
│ Electron    │     │ YooKassa    │     │ Bot         │
└─────────────┘     │ SMS Aero    │     └─────────────┘
                    └─────────────┘
```

### 6.1 Стек (по смете)

| Слой | Технологии |
|------|------------|
| Mobile | React Native, Expo, Redux Toolkit, WebSocket, FCM/APNs, YooKassa SDK |
| Backend | NestJS, PostgreSQL, Redis, Socket.io, Docker |
| PC Launcher | Electron, React, Kiosk Mode |
| Admin | React, Vite, Recharts, WebSocket |
| Bot | Telegraf.js |
| CI/CD | GitHub Actions / аналог, деплой на VPS (Timeweb / Hetzner) |

### 6.2 Основные сущности БД

- `users`, `user_documents`, `sessions` (auth)
- `clubs`, `zones`, `seats`, `hardware_specs`
- `bookings`, `booking_seats`
- `wallets`, `transactions`, `payments`
- `locks`, `lock_events`
- `acceptance_reports`
- `feedback`, `notifications`, `push_tokens`
- `pc_sessions` (связь launcher ↔ booking)

### 6.3 API (группы эндпоинтов)

- `/auth` — OTP, refresh, logout
- `/users` — профиль, история
- `/club` — зоны, места, карта
- `/bookings` — CRUD, calculate, extend
- `/payments` — topup, webhook YooKassa
- `/locks` — open main / open cell
- `/acceptance` — submit checklist
- `/feedback`
- `/admin/*` — защищённые роуты
- WebSocket `/ws` — `seat.status`, `booking.updated`, `session.timer`

## 10. Ограничения и допущения MVP

- Верификация паспорта — опциональный модуль после запуска.
- PC Launcher: визуал и UX финализируются по лучшему референсу с обхода клубов.
- Удалённый рабочий стол — зависит от установки агента/VNC на каждом ПК.
- Один клуб в MVP (мультиклуб — архитектурный задел в `club_id`).
- Сбер Pay / Tinkoff Pay как отдельные SDK — в смете YooKassa; прямую интеграцию уточнить при необходимости.

## 11. Критерии приёмки MVP

- [ ] Регистрация по SMS работает на iOS и Android.
- [ ] Карта обновляет статусы без перезагрузки (< 2 с).
- [ ] Полный цикл: бронь → оплата → таймер → продление.
- [ ] Открытие главной двери и ячейки только в окне брони.
- [ ] Чеклист приёмки сохраняется и виден в админке.
- [ ] PC Launcher: вход по QR/PIN, блокировка после сеанса.
- [ ] Admin: смена статуса ПК отражается на карте клиента.
- [ ] Push о конце сеанса приходит.
- [ ] Приложения опубликованы в Store / Google Play (S-08).
