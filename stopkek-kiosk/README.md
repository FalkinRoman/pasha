# stopkek Kiosk — подложка на ПК (Windows)

Полноэкранное приложение на игровом компьютере. Пока клиент не оплатил сеанс и не отсканировал QR с телефона — рабочий стол заблокирован.

---

## Что нужно на ПК

| Требование | Зачем |
|------------|--------|
| **Windows 10/11** | Целевая ОС |
| **Node.js 20+** | Только для сборки/разработки (`node -v`) |
| **Интернет** | Связь с сервером stopkek |
| Файл **`config.json`** | Номер места + ключ киоска |

После сборки `.exe` Node на игровых ПК **не обязателен** — только сам `stopkek Kiosk.exe` и `config.json` рядом.

---

## Номер ПК (`seatNumber`) — как вести

Номер в `config.json` **должен совпадать** с номером места в **админке** и в **мобильном приложении** при брони.

Пример:
- В админке: [http://46.149.71.92/seats](http://46.149.71.92/seats) — место **№ 5**
- На этом физическом ПК в `config.json`: `"seatNumber": 5`
- Клиент бронирует **место 5** → сканирует QR **с монитора этого ПК**

| ПК в зале | `seatNumber` в config.json |
|-----------|---------------------------|
| Первый у окна | `1` |
| Второй в ряду | `2` |
| … | как в админке |

**На каждом компьютере свой `config.json`** (или один шаблон, но разный `seatNumber`).

Подпись на корпусе: **«ПК #5»** — как в приложении.

## Настройка (один раз на ПК)

### 1. Скопировать проект или `.exe` на Windows

Папка, например: `C:\stopkek-kiosk\`

### 2. Создать `config.json`

Скопируй `config.example.json` → `config.json` в **ту же папку**, что и программа.

```json
{
  "apiUrl": "http://46.149.71.92/api",
  "seatNumber": 5,
  "kioskKey": "ваш-секретный-ключ-одинаковый-на-всех-пк"
}
```

| Поле | Описание |
|------|----------|
| `apiUrl` | Адрес API. Сейчас: `http://46.149.71.92/api`. После домена: `https://stopkek.ru/api` |
| `seatNumber` | Номер этого ПК (см. админку → Места) |
| `kioskKey` | Секретный ключ — **тот же**, что `KIOSK_API_KEY` на сервере |

### 3. Ключ на сервере

На VPS в `deploy/.env.prod` (или `.env` локально):

```env
KIOSK_API_KEY=ваш-секретный-ключ-одинаковый-на-всех-пк
PUBLIC_API_URL=http://46.149.71.92/api
```

После смены — перезапуск API (`docker compose ... up -d --build`).

Миграция БД (один раз на сервере):

```bash
npx prisma migrate deploy
```

## Запуск на Windows

### Вариант A — разработка / тест (нужен Node.js)

```powershell
cd C:\stopkek-kiosk
npm install
npm run dev
```

Откроется полноэкранное окно с QR.

### Вариант B — прод на игровых ПК (рекомендуется)

На машине **с Node.js** (можно твой Mac или один админ-ПК):

```bash
cd stopkek-kiosk
npm install
npm run pack:win
```

Готовый файл: **`release/stopkek Kiosk.exe`** (portable).

На **каждый игровой ПК** скопировать:
```
stopkek Kiosk.exe
config.json          ← свой seatNumber!
```

Запуск: двойной клик по `stopkek Kiosk.exe`.

## Автозапуск при включении Windows

1. `Win + R` → `shell:startup`
2. Ярлык на `stopkek Kiosk.exe` → в папку автозагрузки
3. В свойствах ярлыка «Рабочая папка» = папка, где лежит `config.json`

Либо Планировщик заданий: триггер «При входе в систему», действие — запуск `.exe`.

## Как пользоваться (клиент)

1. На мониторе: **STOPKEK**, QR, «Оплатите в приложении», **ПК #N**.
2. Клиент в мобилке: бронь места **#N** → оплата.
3. Сеанс → **«Сканировать QR на мониторе»** → камера на QR.
4. Экран ПК: таймер, баланс, подсказка продлить в приложении.
5. Время кончилось → снова полный блок «Время вышло».

Вход по SMS/звонку — **только в телефоне**, не на ПК.

## Управление для админа клуба

### Закрыть подложку (обслуживание, игры вне stopkek, настройка)

| Действие | Как |
|----------|-----|
| **Выйти из программы** | **`Ctrl + Shift + Q`** |
| Принудительно | Диспетчер задач → завершить «stopkek Kiosk» |
| Свернуть | Не предусмотрено (режим киоска) |

`Ctrl+Shift+Q` — знать только персоналу, клиентам не говорить.

### Перезапустить

Закрыть (`Ctrl+Shift+Q`) → снова запустить `.exe` или ярлык.

### Сменить номер ПК

1. Закрыть подложку (`Ctrl+Shift+Q`)
2. Править `config.json` → `"seatNumber": новый_номер`
3. Сохранить → запустить снова

### Нет связи с сервером

- Проверить интернет на ПК
- Открыть в браузере: `http://46.149.71.92/api/health` → должно быть `ok`
- Проверить `apiUrl` и `kioskKey` в `config.json`
- На сервере задан ли `KIOSK_API_KEY`

### QR не срабатывает

- Номер в брони = `seatNumber` в `config.json`
- На сервере применена миграция `pcUnlockedAt`
- QR на мониторе обновляется каждые ~2 мин — отсканировать свежий
- В мобилке активный оплаченный сеанс
